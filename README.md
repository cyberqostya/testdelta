# testdelta

## Установка

`npm i`  
`npm run dev`

## Заметки

1. Семантически верстка таблицы
2. Вынесены отдельно scss переменные для переиспользуемости
3. Данные с сервера  
   3.1. генерировались с помощью функции /utils/getRandomData (но с сохранением исходных по ТЗ) с максимальным разбросом в 40% от данных текущего дня для наглядности графика  
   3.2. берутся путем фейкового ajax запроса к backend'у  
   3.3 конвертируются в "отображаемые данные" через функцию convertServerDataToDispayedData, чтобы рассчеты производились единожды и не прямиком в верстке
4. Окрас ячеек столбца "Вчера" рассчитывается автоматически (не хардкодно) на основании процента от значения данных ячейки "Текущий день" той же строки
5. Последний столбец таблицы ("Этот день недели" из ТЗ) принял как среднее арифметическое и окрашивается случайным образом (как в ТЗ), так как не смог логически его связать с данными
6. Использована библиотека highcharts-vue для построения графиков зависимости данных от дней недели  
   6.1 Данные графика строятся по данным с сервера, где предпоследние и последние данные это за "Вчера" и "Текущий день" соответственно
7. Плавная анимация раскрытия графика с данными и наведение на строку данных
8. Мобильная версия реализована просто - стилизован скроллбар (с прозрачным запасом по высоте) по оси X после преодоления breakpoint'а
9. Далее должна следовать работа по оптимизации при, скажем, 1000 строк таблицы, которые пришли с сервера, но в данной работе её опустил. Как минимум можно не отрисовывать графики пока они не будут открыты пользователем.
